# Source Policy

- При конфликте между устным описанием и файлами репозитория приоритет у согласованной документации в репозитории.
- Если кодовый репозиторий доступен, его можно использовать только как вспомогательный read-only источник для implementation verification.
- Неподтверждённые предположения должны маркироваться как `Неизвестно / требует уточнения`.
