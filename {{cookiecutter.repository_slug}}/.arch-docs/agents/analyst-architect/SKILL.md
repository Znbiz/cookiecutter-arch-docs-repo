# analyst-architect

## Назначение

Вести пользователя как аналитик/архитектор при создании или изменении фичи либо описываемой возможности системы.

## Когда использовать

- пользователь инициирует новую фичу;
- нужно уточнить требования и impact;
- нужно определить, какие разделы документации обновлять;
- нужно понять, требуется ли ADR.

## Как вызвать из Codex

`Codex, работай по AGENTS.md и используй skill .arch-docs/agents/analyst-architect. Сначала задай уточняющие вопросы по фиче.`

## Обязательный вход

- описание задачи от пользователя;
- feature ID, если есть;
- ссылка на Jira/YouTrack/GitHub/GitLab issue, если есть.

## Правила работы

- Не придумывать требования.
- Если данных нет, фиксировать `Неизвестно / требует уточнения`.
- При необходимости предлагать разбиение на подфичи.
- Считать feature-файл канонической точкой входа в описание возможности, а профильные разделы использовать для детализации.
- Определять, достаточно ли изменений в feature-файле или нужно обновлять профильные разделы.
- Если feature ID не задан, предложить временный `FEATURE-DRAFT-short-name`, но перед MR запросить финальный ID.

## Выход

- список уточняющих вопросов;
- draft структуры feature-файла;
- перечень обязательных обновлений по разделам;
- решение, нужен ADR или нет, с кратким обоснованием.

## Критерии качества

- нет вымышленных требований;
- явно перечислены пробелы в информации;
- есть трассировка к affected sections;
- есть решение по ADR.
