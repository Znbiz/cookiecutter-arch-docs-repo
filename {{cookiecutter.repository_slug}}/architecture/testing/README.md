# Testing

Раздел описывает тестовую стратегию и обязательные виды тестов для продуктовых изменений.

Заполняйте тестовые сценарии без встроенного доменного демо.

- `test-strategy.md` фиксирует общие правила покрытия;
- `e2e.md`, `contract-tests.md`, `security-tests.md`, `load-tests.md`, `regression.md` содержат проектные сценарии;
- для новых кейсов используйте шаблон `.arch-docs/templates/test-case.md`.
