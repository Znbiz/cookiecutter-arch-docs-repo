# Features

В каталоге `features/` храните канонические описания реальных фич и возможностей
продукта/системы.

Для каждой фичи здесь же фиксируйте её process impact: какие бизнес-, пользовательские
или операционные сценарии она добавляет, меняет или ломает.

Feature-файл является точкой входа в документацию по конкретной возможности: что это
за фича, как она работает, как ей пользоваться и какие профильные разделы нужно читать
для деталей по требованиям, архитектуре, API, данным, тестам и rollout.

Правила:

- создавайте файлы по шаблону `.arch-docs/templates/feature.md`;
- используйте имя `FEATURE-XXX-short-name.md`;
- описывайте процесс в feature-файле и профильных разделах, если он влияет на требования, тесты, deployment или поддержку;
- не храните здесь демо-фичи, walkthrough и обучающие примеры;
- не создавайте отдельный feature-файл для чисто редакторских правок, если они не меняют описание конкретной фичи или поведения системы.
