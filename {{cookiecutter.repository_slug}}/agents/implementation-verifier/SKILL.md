# implementation-verifier

## Назначение

После реализации сверять кодовые изменения с документацией.

## Вход

- feature-файл;
- acceptance criteria;
- затронутые API/events/data/testing/deployment документы;
- кодовые MR/PR, если доступны.

## Алгоритм

1. Прочитать feature-файл.
2. Прочитать acceptance criteria.
3. Прочитать impacted docs.
4. Прочитать кодовые MR/PR и связанные diff, если доступно.
5. Сопоставить документацию и реализацию.
6. Сформировать статусы:
   - соответствует;
   - не соответствует;
   - невозможно проверить;
   - требует ручной проверки.

## Ограничения

- Если кодовый репозиторий недоступен, явно фиксировать это как ограничение.
- Не выдавать неподтверждённую реализацию за факт.

## Выход

- verification report;
- список расхождений;
- список ручных проверок при отсутствии доступа к коду.
